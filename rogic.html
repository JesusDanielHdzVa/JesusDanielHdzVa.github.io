¿Rogic?

¿Que es?

stroRogic (Rule-based Open Graph Interactive Computing) es un lenguaje de programación y un sistema basado en reglas para el procesamiento de datos y la inteligencia artificial. Se enfoca en la representación del conocimiento y la inferencia lógica a través de grafos abiertos, lo que lo hace útil en áreas como la IA simbólica, la toma de decisiones y la automatización de procesos.

¿Porque Aprenderlo?

Ideal para IA simbólica: Facilita la representación y manipulación del conocimiento en aplicaciones de inteligencia artificial.

Automatización de reglas: Permite definir reglas lógicas para automatizar procesos y tomar decisiones basadas en datos estructurados.

Interoperabilidad con grafos: Se integra bien con sistemas que usan bases de datos gráficas como Neo4j.

Útil en razonamiento automático: Aplicable en sistemas expertos, análisis de datos y generación de inferencias.

Componentes de la Interfaz de Rogic

La interfaz de Rogic está diseñada para ofrecer una experiencia intuitiva y eficiente. A continuación, se describen en detalle seis componentes clave del entorno:

1. Botones de Borrar Comandos

Estos botones permiten eliminar o limpiar los comandos previamente ingresados en el área de comandos. Son esenciales para borrar errores o reiniciar una secuencia de instrucciones sin tener que cerrar el proyecto. Generalmente, cuentan con iconos identificativos, como una "X" o un borrador, y están ubicados de forma accesible para facilitar una edición rápida.

2. Área de Comandos

El Área de Comandos es el espacio destinado a la entrada de instrucciones o comandos específicos. Aquí se pueden escribir y modificar secuencias de comandos, permitiendo una interacción directa con el entorno de desarrollo. Este espacio suele incluir funcionalidades como historial de comandos y autocompletado para mejorar la productividad.

3. Barrera de Navegación

La Barrera de Navegación actúa como un delimitador visual que separa distintas secciones del entorno, facilitando la orientación del usuario. Esta división ayuda a identificar claramente dónde se encuentra el área de programación, el área de comandos y otras secciones del IDE, manteniendo una estructura organizada y evitando confusiones.

4. Área de Programación

El Área de Programación es el núcleo del entorno, donde se escribe, edita y visualiza el código fuente. Este espacio cuenta con características como resaltado de sintaxis, numeración de líneas y, en algunos casos, sugerencias de código. Su diseño está optimizado para minimizar distracciones y permitir al usuario concentrarse en el desarrollo del programa.

5. Menú de Opciones

El Menú de Opciones ofrece acceso a diversas configuraciones y herramientas del entorno. Desde este menú se pueden ajustar preferencias personales, cambiar el tema del editor, configurar el compilador o acceder a funciones avanzadas de depuración y ejecución. Es esencial para personalizar la experiencia y adaptar el entorno a las necesidades del usuario.

6. Área de Botones

El Área de Botones agrupa una serie de botones de acceso rápido que permiten ejecutar funciones comunes sin tener que navegar por menús complejos. Estos botones incluyen acciones como compilar, ejecutar, guardar y formatear el código, ayudando a agilizar el flujo de trabajo y facilitando la interacción con el entorno.

Operadores en Rogic

Los operadores aritméticos en Rogic permiten realizar operaciones matemáticas básicas entre variables o valores numéricos. Son fundamentales para la mayoría de los programas, ya que permiten manipular y procesar datos de forma lógica. A continuación se describen los principales operadores y su función.

1. Suma (+)

El operador de suma se utiliza para agregar dos valores numéricos. Por ejemplo, si a = 3 y b = 5, la expresión a + b dará como resultado 8. Este operador también puede utilizarse para concatenar cadenas de texto en algunos contextos.

2. Resta (-)

Este operador permite restar un valor de otro. Por ejemplo, si a = 10 y b = 4, entonces a - b devuelve 6. Es útil para calcular diferencias entre cantidades.

3. Multiplicación (*)

El operador de multiplicación se usa para obtener el producto de dos números. Si a = 6 y b = 7, entonces a * b resulta en 42. Es esencial en programas que trabajan con fórmulas matemáticas o repeticiones.

4. División (/)

Este operador divide un número entre otro. Por ejemplo, 20 / 4 devolverá 5. Es importante considerar que si se divide entre cero, se generará un error, por lo que se recomienda validar los valores antes de usar este operador.

5. Módulo (%)

El operador módulo devuelve el residuo de una división. Por ejemplo, 17 % 5 da como resultado 2, ya que 5 cabe 3 veces en 17 con un sobrante de 2. Es útil para verificar si un número es par o impar, o para ciclos de repetición con condiciones específicas.

Operadores Relacionales en Rogic

Los operadores relacionales en Rogic permiten comparar valores entre sí. Su propósito principal es devolver un resultado lógico, es decir, true (verdadero) o false (falso), dependiendo de si la condición evaluada se cumple. Son ampliamente usados en estructuras condicionales y bucles.

1. Igualdad (==)

Este operador compara si dos valores son exactamente iguales. Por ejemplo, 5 == 5 devolverá true, mientras que 5 == 3 devolverá false. Es común en sentencias IF y otras condiciones lógicas.

2. Desigualdad (!=)

Sirve para comprobar si dos valores son diferentes. Por ejemplo, 8 != 5 es true, pero 8 != 8 es false. Es útil para ejecutar acciones si los valores no coinciden.

3. Mayor que (>)

Evalúa si el valor de la izquierda es mayor que el de la derecha. Por ejemplo, 10 > 4 es true. Si se quiere saber si un número excede a otro, este es el operador indicado.

4. Menor que (<)

Compara si el valor de la izquierda es menor que el de la derecha. Por ejemplo, 2 < 9 devuelve true. Es común en validaciones de rangos mínimos.

5. Mayor o igual que (>=)

Verifica si un valor es mayor o igual a otro. Por ejemplo, 6 >= 6 devuelve true, lo que lo hace útil para condiciones inclusivas de máximos.

6. Menor o igual que (<=)

Determina si un valor es menor o igual al otro. Por ejemplo, 4 <= 7 o 4 <= 4 devuelven true. Se usa en validaciones de límites inferiores.

Estos operadores permiten que los programas puedan tomar decisiones en función de las comparaciones entre datos, lo cual es esencial para el control de flujo y la lógica de cualquier programa en Rogic.

Uso de Condicionales en Rogic para Robots

Los condicionales en Rogic permiten que el robot tome decisiones según ciertas condiciones. A continuación se muestran ejemplos prácticos:

Condicional IF ELSE

IF (Si se cumple una condición): El robot ejecuta una acción si una condición es verdadera.

// Si el sensor de luz detecta oscuridad, enciende el LED IF SENSOR_LUZ < 300 THEN ENCENDER_LED ENDIF

IF-ELSE (Si se cumple, hacer esto; si no, hacer otra cosa): El robot elige entre dos caminos según la condición.

// Si el botón está presionado, el robot avanza; si no, se detiene IF BOTON == PRESIONADO THEN MOVER_ADELANTE ELSE DETENER ENDIF

IF - ELSE IF - ELSE: Permite evaluar múltiples condiciones en orden.

// El robot ajusta la velocidad según la distancia IF DISTANCIA < 10 THEN DETENER ELSEIF DISTANCIA < 20 THEN VELOCIDAD = 50 ELSE VELOCIDAD = 100 ENDIF

Condicional anidado: Se pueden colocar condiciones dentro de otras.

// Si hay luz y el botón está presionado, el robot gira a la izquierda IF SENSOR_LUZ > 500 THEN IF BOTON == PRESIONADO THEN GIRAR_IZQUIERDA ENDIF ENDIF

Uso de sensores con condiciones: Los sensores son clave en decisiones del robot.

// Si el sensor de línea detecta una línea negra, el robot gira IF SENSOR_LINEA == NEGRO THEN GIRAR_DERECHA ENDIF

Uso de Variables en Rogic para Robots

Las variables en Rogic permiten guardar información que el robot puede usar, cambiar o evaluar más adelante. Son útiles para contar, tomar decisiones o ajustar comportamientos según el entorno.

Declaración y asignación de variables

Crear una variable: Puedes declarar una variable y asignarle un valor inicial.

// Crear una variable llamada "contador" con valor inicial 0
SET contador = 0

Modificar el valor de una variable: Puedes cambiar el valor de una variable durante la ejecución.

// Aumentar el valor de "contador" en 1
contador = contador + 1

Usar variables en condicionales: Las variables se pueden evaluar para decidir qué acción tomar.

// Si el contador es mayor que 5, encender un LED
IF contador > 5 THEN
  ENCENDER_LED
ENDIF

Variables con sensores: Puedes guardar valores de sensores en variables para analizarlos luego.

// Guardar el valor del sensor de luz en una variable
SET luz = SENSOR_LUZ
// Si hay poca luz, encender el LED
IF luz < 300 THEN
  ENCENDER_LED
ENDIF

Variables acumulativas: Sirven para contar eventos o sumar valores a lo largo del tiempo.

// Contar cuántas veces se presiona un botón
IF BOTON == PRESIONADO THEN
  contador = contador + 1
ENDIF

Temporizador simple con variables: Usar una variable como contador de tiempo.

Más ejemplos del uso de variables en Rogic

// Crear una variable llamada "tiempo" y establecerla en 0
SET tiempo = 0
// Aumentar "tiempo" cada vez que pasa un segundo
tiempo = tiempo + 1
// Si "tiempo" es igual a 10, reproducir un sonido
IF tiempo == 10 THEN
  REPRODUCIR_SONIDO
ENDIF

Comparación entre dos variables: Se pueden comparar valores almacenados para tomar decisiones.

// Comparar dos sensores y actuar según cuál tenga más valor
SET izquierda = SENSOR_IZQUIERDO
SET derecha = SENSOR_DERECHO
IF izquierda > derecha THEN
  GIRAR_DERECHA
ELSE
  GIRAR_IZQUIERDA
ENDIF

Variable como bandera (flag): Para activar o desactivar algo.

// Variable "activo" indica si el sistema está encendido
SET activo = TRUE
IF activo == TRUE THEN
  MOVER_ADELANTE
ENDIF

Condición múltiple usando variables: Verifica varias condiciones al mismo tiempo.

// Si hay luz y la distancia es segura, avanzar
SET luz = SENSOR_LUZ
SET distancia = SENSOR_ULTRASONICO
IF luz > 400 AND distancia > 15 THEN
  MOVER_ADELANTE
ELSE
  DETENER
ENDIF

Reinicio de variables: Puedes resetear variables cuando sea necesario.

// Si se presiona el botón "reset", reiniciar el contador
IF BOTON_RESET == PRESIONADO THEN
  contador = 0
ENDIF

Contador de ciclos o vueltas: útil para tareas repetitivas o seguimiento de repeticiones.

// Contar cuántas veces el robot gira
SET vueltas = 0
// Cada vez que detecta una vuelta completa
IF SENSOR_VUELTA == ACTIVADO THEN
  vueltas = vueltas + 1
ENDIF

Entrada y Salida de Datos en Rogic

Los robots utilizan entradas para obtener información del entorno (como sensores) y salidas para realizar acciones (como luces, motores o sonidos).

Ejemplos usando el buzzer como salida

Usar un sensor como entrada para activar el buzzer: El robot recibe un dato del sensor y responde con un sonido.

// Si el sensor de distancia detecta algo cerca, activar el buzzer
IF SENSOR_ULTRASONICO < 10 THEN
  ENCENDER_BUZZER
ENDIF

Desactivar el buzzer cuando ya no se cumpla la condición:

// Si la distancia es mayor o igual a 10, apagar el buzzer
IF SENSOR_ULTRASONICO >= 10 THEN
  APAGAR_BUZZER
ENDIF

Entrada combinada: botón + sensor: Usar más de una entrada antes de emitir una salida.

// Si el botón está presionado y hay oscuridad, sonar el buzzer
IF BOTON == PRESIONADO AND SENSOR_LUZ < 300 THEN
  ENCENDER_BUZZER
ENDIF

Salida temporal con buzzer: Hacer sonar el buzzer por un tiempo determinado.

// Si se presiona el botón, sonar el buzzer por 2 segundos
IF BOTON == PRESIONADO THEN
  ENCENDER_BUZZER
  ESPERAR 2 SEGUNDOS
  APAGAR_BUZZER
ENDIF

Activar múltiples salidas a la vez: Usar más de una salida como respuesta.

// Si se detecta una línea negra, sonar el buzzer y encender LED
IF SENSOR_LINEA == NEGRO THEN
  ENCENDER_BUZZER
  ENCENDER_LED
ENDIF

Usar una variable para decidir si se activa el buzzer: Entrada → variable → salida.

// Guardar el valor del sensor de gas
SET gas = SENSOR_GAS
// Si el valor del gas supera el límite, activar buzzer
IF gas > 80 THEN
  ENCENDER_BUZZER
ELSE
  APAGAR_BUZZER
ENDIF

Más ejemplos de entrada y salida con buzzer

Buzzer de advertencia cuando se supera un límite de temperatura:

// Si la temperatura supera los 35 grados, sonar buzzer
SET temperatura = SENSOR_TEMPERATURA
IF temperatura > 35 THEN
  ENCENDER_BUZZER
ENDIF

Respuesta del buzzer ante múltiples condiciones peligrosas:

// Si hay humo o gas, activar alarma con buzzer
IF SENSOR_HUMO == DETECTADO OR SENSOR_GAS > 100 THEN
  ENCENDER_BUZZER
ENDIF

Buzzer intermitente para señal de emergencia:

// Si se activa una alerta, hacer sonar el buzzer dos veces
IF ALERTA == ACTIVADA THEN
  ENCENDER_BUZZER
  ESPERAR 1 SEGUNDO
  APAGAR_BUZZER
  ESPERAR 1 SEGUNDO
  ENCENDER_BUZZER
  ESPERAR 1 SEGUNDO
  APAGAR_BUZZER
ENDIF

Control manual del buzzer con interruptor:

// Encender o apagar el buzzer según el estado de un interruptor
IF INTERRUPTOR == ON THEN
  ENCENDER_BUZZER
ELSE
  APAGAR_BUZZER
ENDIF

Confirmación sonora con buzzer al presionar un botón:

// Sonar el buzzer como respuesta al presionar un botón
IF BOTON == PRESIONADO THEN
  ENCENDER_BUZZER
  ESPERAR 0.5 SEGUNDOS
  APAGAR_BUZZER
ENDIF

¿Qué es el Editor de Rogic?

El Editor de Rogic es una herramienta de programación visual diseñada para controlar robots de manera sencilla. Utiliza bloques que los usuarios pueden arrastrar y combinar para crear secuencias de instrucciones. Cada bloque representa una acción o condición, y se usan para programar comportamientos del robot de forma intuitiva.

Los robots en Rogic pueden recibir entradas (como sensores o botones) y realizar salidas (como encender LEDs, mover motores, o emitir sonidos con el buzzer). El uso de entradas y salidas es clave para interactuar con el entorno y controlar el robot.

Esta plataforma es ideal para principiantes, ya que permite comprender los conceptos de programación sin necesidad de escribir código, y visualiza la lógica detrás de cada acción.

Ejemplos de Entrada y Salida usando Sensores, LEDs, Motores y Buzzer

1. Usar el sensor ultrasónico para activar un motor

Si la distancia detectada por el sensor ultrasónico es menor a 10 cm:
- Encender motor (avanzar)
Si la distancia es mayor o igual a 10 cm:
- Detener motor

2. Activar un LED cuando se presiona un botón

Si el botón está presionado:
- Encender LED
Si el botón no está presionado:
- Apagar LED

3. Usar un sensor de luz para controlar la velocidad del motor

Si el sensor de luz detecta un valor bajo (oscuro):
- Establecer velocidad del motor en 50%
Si el sensor de luz detecta un valor alto (luz):
- Establecer velocidad del motor en 100%

4. Buzzer y LED como alerta para detección de gas

Si el sensor de gas detecta un valor mayor a 80:
- Encender buzzer
- Encender LED rojo
Si el valor es menor o igual a 80:
- Apagar buzzer
- Apagar LED

5. Controlar un motor con el interruptor

Si el interruptor está en ON:
- Encender motor (avanzar)
Si el interruptor está en OFF:
- Detener motor

6. Combinar sensores para activar múltiples salidas (buzzer y motor)

Si el sensor de distancia es menor a 5 cm y el sensor de línea detecta una línea negra:
- Encender buzzer
- Detener motor
Si no:
- Apagar buzzer
- Avanzar motor

7. Hacer sonar el buzzer intermitentemente cuando se detecta una línea negra

Si el sensor de línea detecta una línea negra:
- Encender buzzer
- Esperar 0.5 segundos
- Apagar buzzer
- Esperar 0.5 segundos

8. Hacer avanzar el robot si el sensor de temperatura está por debajo de 30 grados

Si el sensor de temperatura está por debajo de 30 grados:
- Encender motor (avanzar)
Si el sensor de temperatura supera los 30 grados:
- Detener motor

9. Controlar la velocidad de un motor dependiendo de la proximidad a un objeto

Si el sensor ultrasónico detecta un objeto a menos de 5 cm:
- Establecer velocidad del motor en 20%
Si el objeto está a más de 5 cm:
- Establecer velocidad del motor en 100%

10. Usar múltiples condiciones para encender un LED y buzzer

Si el sensor de luz detecta oscuridad y el sensor de temperatura es mayor a 30 grados:
- Encender LED rojo
- Encender buzzer
Si no:
- Apagar LED
- Apagar buzzer
